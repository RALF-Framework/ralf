# Contributing to RALF

Thank you for your interest in RALF.

RALF is an early-stage framework. Contributions should help clarify the model, ground it in existing standards, and make it more useful for real organizations.

## Contribution principles

1. **Reuse before inventing.** If a standard or existing practice already solves a problem, prefer reusing it.
2. **Keep domain experts in mind.** RALF should be understandable by people who know their domain but may not know AI or software architecture.
3. **Keep humans accountable.** Do not propose designs that hide ownership, approval, or responsibility.
4. **Make artifacts explicit.** Work should produce evidence that can be reviewed and traced.
5. **Separate framework from product.** This repository defines the open framework, not every feature of RALF Studio.

## Good contribution types

- clearer definitions
- examples from real workflows
- domain pack proposals
- schema suggestions
- governance improvements
- source/foundation improvements
- diagrams
- documentation fixes

## Before proposing a new concept

Please answer:

1. What problem does this solve?
2. Which existing standard or practice is related?
3. Why is the existing standard not enough by itself?
4. How does this fit into lifecycle, role, artifact, gate, trace, or context packet concepts?
5. What human accountability or approval requirement applies?

## Documentation style

Use:

- simple language
- concrete examples
- short sections
- diagrams where helpful
- stable external references

Avoid:

- hype language
- unexplained acronyms
- agent-first thinking
- vendor lock-in
- vague claims about automation

## Pull requests

For early-stage contributions:

- keep pull requests small
- explain the reasoning
- reference relevant standards or sources
- include examples when adding concepts
- update related docs when terminology changes

## Code of conduct

Be respectful, constructive, and specific. The goal is to create a useful framework for many domains and many types of contributors.
